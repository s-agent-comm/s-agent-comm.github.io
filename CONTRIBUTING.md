# How to Contribute

We welcome contributions from all members of the community! This document outlines how you can participate and help improve the Semantic Agent Ontology. Our primary goal right now is to gather feedback and build a strong community around our [W3C Community Group Proposal](https://www.w3.org/community/groups/proposal/...).

## Getting Started

The best way to join the effort is to get involved in the conversation.

1.  **Join the W3C Community Group:** This is the central hub for all discussions. [**Click here to join and participate.**](https://www.w3.org/community/blog/2025/11/09/proposed-group-semantic-agent-communication-community-group/)
2.  **Familiarize yourself with the project:** Read the `README.md` and `specs/Explainer.md` to understand the project's goals and technical foundations.
3.  **Review our proposal:** Read our [W3C Community Group proposal](https://www.w3.org/community/blog/2025/11/09/proposed-group-semantic-agent-communication-community-group/) and see the vision we've laid out.

## How to Provide Feedback

Your feedback on the proposal is critical at this stage. You can:

*   **Use GitHub Issues:** Open an issue to provide feedback, ask questions, or suggest changes related to the proposal. Please tag it with the `w3c-proposal` label.
*   **Participate in the W3C Mailing List:** Once you join the community group, you will have access to the mailing list where official discussions happen.

### Contributing Use Cases and Examples

All use cases and practical examples are now managed in a dedicated repository. If you wish to contribute a new use case or example, please visit the **[Agent Ontology Use Cases repository](https://github.com/s-agent-comm/agent-ontology-use-cases)** and follow its contribution guidelines.

## Making Contributions via Pull Requests

We actively welcome your pull requests for concrete changes.

### W3C Community Group Specifics

As a W3C Community Group, all contributions to this repository are subject to the [W3C Community Contributor License Agreement (CLA)](https://www.w3.org/community/about/agreements/cla/). By contributing, you agree to the terms of this agreement. Please familiarize yourself with the [W3C Community Group Process Document](https://www.w3.org/community/about/process/) for a full understanding of how decisions are made, which is primarily through consensus among active participants.

1.  **Fork the repository:** Start by forking the main repository to your own GitHub account.
2.  **Create a new branch:** For any new feature or fix, please create a new branch in your fork.
    ```bash
    git checkout -b my-new-feature
    ```
3.  **Make your changes:** Make your changes to the ontology, schemas, or documentation.
4.  **Commit your changes:** Write a clear and concise commit message.
    ```bash
    git commit -am "feat: Add new property for Capability"
    ```
5.  **Push to your branch:**
    ```bash
    git push origin my-new-feature
    ```
6.  **Open a Pull Request:** Open a pull request from your branch to the main repository's `main` branch. In the description, please explain the "why" and "what" of your changes, and link to any relevant issues.

## Code of Conduct

This project and everyone participating in it is expected to follow the [W3C Code of Ethics and Professional Conduct](https://www.w3.org/Consortium/cepc/). Please be respectful and constructive in all your interactions.
